## Q7. List and explain different Project management approaches

| Approach | Explanation |
|----------|-------------|
| **1. Traditional** | Planning, executing, controlling tasks to reach goal under budget and within timeframe. Best when requirements are clear and changes minimal. |
| **2. Agile** | Highly collaborative methodology implementing feedback for flexibility and continuous improvement. Emphasizes: small batches, customer collaboration, quick feedback, transparency. Best for software projects. |
| **3. Waterfall** | Sequential linear steps; one phase completes before next starts. Used in construction, manufacturing, well-defined projects. |
| **4. Systems Approach** | Visualizes and analyzes physical/conceptual systems. Framework for conceptualizing problems, solving problems, making decisions. |
| **5. Critical Chain / Critical Path** | **Critical Path:** Longest sequence of tasks determining project duration. **Critical Chain:** Sequence of precedence and resource-dependent tasks. |
| **6. Program Management** | For handling multiple projects simultaneously; used for change management, industrial engineering. |

## Q8. ‘Scope’ and ‘Feasibility’ are two important and significant concepts in project management. Explain the statement in detail

### Scope:

| Aspect | Explanation |
|--------|-------------|
| **Definition** | The sum of products, services, and results to be provided as a project. Defines boundaries, deliverables, tasks. |
| **Importance** | Prevents scope creep, sets clear expectations, determines resources needed, provides baseline for tracking progress. |
| **Components** | Scope planning, scope definition, scope verification, scope control. |

### Feasibility:

| Aspect | Explanation |
|--------|-------------|
| **Definition** | Systematic evaluation of a proposed project's potential success and viability before committing resources. |
| **Types** | Technical feasibility, operational feasibility, economic feasibility, legal feasibility, schedule feasibility. |
| **Importance** | Assesses if project is realistically achievable within time, budget, resource constraints; identifies risks early; determines whether to proceed. |

### How they are related:

- **Scope defines WHAT** needs to be done.
- **Feasibility determines IF** it can be done (technically, economically, operationally).
- Changes in scope affect feasibility – adding features may make project technically or financially unfeasible.
- Feasibility study helps validate and refine scope before commitment.

## Q9. Elaborate on systems approach in project management with suitable example

### Definition:
> *"A way to visualize and analyze physical and conceptual systems – a framework for conceptualizing problems, solving problems, and making decisions."*

### Components of a System:

| Component | Description |
|-----------|-------------|
| **Sub-System** | Operating level; departments are subsystems of a company |
| **Environment (Supra System)** | External factors affecting the system (economic, legal, political, social) |
| **System Boundary** | Outer limit of system's operations |
| **Input-Output Conversion** | Importing inputs, transforming them through processes, exporting outputs |
| **Feedback** | Information about system's status and performance |
| **Synergy** | Combined output > sum of individual outputs (2+2=5) |

### Example:
> **Construction of a Metro Project**
> - **Sub-systems:** Design team, construction team, procurement team, finance team, safety team
> - **Environment:** Government regulations, weather, public opposition, funding agencies
> - **Inputs:** Raw materials, labor, equipment, money, information
> - **Conversion process:** Excavation, tunneling, laying tracks, installing electrical systems
> - **Outputs:** Operational metro line, reduced traffic congestion
> - **Feedback:** Progress reports, quality checks, stakeholder reviews

The systems approach ensures all sub-systems work together towards the common goal.

## Q11. Sketch and explain the common stretched S-project life cycle

*(Sketch description)*

The **stretched S-curve** shows cumulative effort/cost over project time:

```
Cumulative
Cost/Effort
     ▲
     │                    ╱
     │                  ╱
     │                ╱
     │              ╱
     │            ╱
     │          ╱
     │        ╱
     │      ╱
     │    ╱
     │  ╱
     │╱
     └──────────────────────► Time
```

### Explanation:

| Segment | Phase | Characteristics |
|---------|-------|-----------------|
| **Flat start (slow rise)** | Initiation/Concept | Low resource usage, planning, feasibility |
| **Steep middle (rapid rise)** | Development/Execution | High resource consumption, main work |
| **Flat end (tapering)** | Closure/Transfer | Winding down, testing, handover |

### Why "stretched"?
The end portion stretches because final tasks (documentation, testing, client acceptance) often take longer than expected – team members may "drag their feet," and many parts must come together.

## Q12. Describe Triple constraints of project management

### The Three Major Constraints:

```
                    ┌─────────────┐
                    │    SCOPE    │
                    │  (What)     │
                    └──────┬──────┘
                           │
          ┌───────────────┼───────────────┐
          │               │               │
          ▼               ▼               ▼
    ┌─────────┐     ┌─────────┐     ┌─────────┐
    │  TIME   │     │ PROJECT │     │  COST   │
    │ (When)  │     │  GOAL   │     │ (How    │
    │         │     │         │     │  much)  │
    └─────────┘     └─────────┘     └─────────┘
                           │
                    ┌──────┴──────┐
                    │   QUALITY   │
                    │ (4th const.)│
                    └─────────────┘
```

| Constraint | Description |
|------------|-------------|
| **Scope** | What must be delivered; features, functionalities, goals |
| **Time** | Schedule for completion; deadlines, milestones |
| **Cost** | Budget allocated; resources, tools, salaries, overheads |
| **Quality** (4th constraint) | Ensures project meets standards and satisfies user expectations |

### Interrelationship:
- Change in one constraint affects others
- **Increase scope** → requires more time and cost, may affect quality
- **Reduce budget** → may limit resources, affecting time and quality
- **Tight deadlines** → may lead to shortcuts, reducing quality

## Q13. “In project management Cost is function of Performance, Time and Scope” Explain with suitable example

### Explanation:

> **Cost = f (Performance, Time, Scope)**

| Relationship | Effect |
|--------------|--------|
| **Cost vs. Scope** | Increasing scope (adding features) → increases cost |
| **Cost vs. Time** | Shortening time → increases cost (overtime, extra resources) |
| **Cost vs. Performance (Quality)** | Higher quality → increases cost (skilled labor, better materials, more testing) |

### Example:

**Software Development Project**

Client requirements:
- Initial: Basic e-commerce website (scope) in 6 months (time) with standard quality (performance)
- Budget: ₹50 lakhs

Changes requested:
1. Add payment gateway integration (+ scope)
2. Deliver in 4 months instead of 6 (- time)
3. Ensure 99.9% uptime and security certification (+ performance)

**Result:** Cost increases to ₹85 lakhs due to:
- Additional developers (overtime)
- Security consultants and testing
- Expedited procurement

Thus, any change in scope, time, or performance directly affects cost.

## Q16. List and explain five criteria Cleland and King have suggested for determining when to use project management

Cleland and King's criteria for determining when to use project management:

| Criterion | Explanation |
|-----------|-------------|
| **1. New or unique task** | When the work is non-repetitive and has not been done before, project management provides necessary structure. |
| **2. Complex task requiring integration** | When multiple functions (engineering, marketing, finance) must coordinate, project management integrates them. |
| **3. Task with high risk or uncertainty** | When outcomes are uncertain, project management's planning and risk management processes help. |
| **4. Task with specific time, cost, and performance constraints** | When there are strict deadlines, budgets, and quality requirements, project management ensures control. |
| **5. Task involving multiple stakeholders** | When many parties have interest in the outcome, project management facilitates communication and expectation management. |
