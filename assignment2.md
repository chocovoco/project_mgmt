## Q14. Discuss the relative need for both technical and managerial competence for project manager

A project manager needs **both technical and managerial competence** for successful project delivery.

| Competence | Need | Examples |
|------------|------|----------|
| **Technical Competence** | To evaluate technical concepts, communicate with team, assess risks, make trade-offs between cost, schedule, and technical issues | Understanding technology, engineering tools, customer requirements, product applications, evolving trends |
| **Managerial Competence** | To plan, organize, lead, control resources, manage stakeholders, resolve conflicts, make decisions | Planning, budgeting, scheduling, team building, communication, negotiation, conflict resolution |

### Why both are essential:

| Technical alone | Managerial alone | Both together |
|----------------|------------------|---------------|
| Cannot manage people, budget, schedule | Cannot evaluate technical feasibility | Complete project delivery |
| Poor stakeholder communication | Poor risk assessment on technical aspects | Balanced decisions |
| Team may lack direction | Technical team may not respect non-technical manager | Team trust and efficiency |

**Conclusion:** For high-technology projects, technical qualifications (engineering degree) are important; for all projects, managerial skills are equally critical. The PM must be a **generalist** with enough technical depth to lead.

## Q17. Describe ‘Project Office’

### Definition:
The project office (also called Project Management Office – PMO) is a support staff group that reports to the project manager, and also a physical place where the project team meets.

### Functions/Responsibilities:

| Function | Description |
|----------|-------------|
| **Center of information** | For internal and external stakeholders |
| **Follow contract requirements** | Control time, cost, performance |
| **Document tasks** | Communicate to all stakeholders |
| **Ensure authorization** | Verify tasks accomplished and their funding |
| **Integrate work** | Across functional departments |

### Composition:
- Depends on authority of project manager, size, importance, and goal of project
- When coordinating multiple projects, staff is larger – called **Project Management Office (PMO)**

## Q18. Compare project manager with functional manager

| Aspect | Project Manager | Functional Manager |
|--------|----------------|---------------------|
| **Focus** | Specific project goals, deliverables | Departmental goals, ongoing operations |
| **Authority** | Over project resources temporarily | Over permanent departmental resources |
| **Reporting** | Team members report to PM for project work | Employees report permanently |
| **Time horizon** | Temporary (project life cycle) | Permanent (ongoing) |
| **Responsibility** | Complete project within time, cost, scope | Maintain technical excellence, resource pool |
| **Decision-making** | Project-specific decisions | Departmental policies and procedures |
| **Conflict resolution** | Handles cross-functional conflicts within project | Handles intra-departmental conflicts |
| **Performance measure** | Project success (on time, budget, quality) | Departmental efficiency, resource utilization |

## Q19. Explain ‘integration in Large scale projects’ (Mega projects)

### Definition:
> *"Project Integration Management includes processes to identify, define, combine, unify, and coordinate various processes and project management activities."* – PMBOK

### Characteristics of Mega Projects:
- Cost > $1 billion USD
- Take many years to build
- Multiple public and private stakeholders
- Transformational, impact millions of people

### Four Dimensions of Success for Mega Projects:

| Dimension | Question |
|-----------|----------|
| **Efficiency** | Did it fulfill cost, time, productivity goals? |
| **Impact on customers/users** | Did it meet user needs? |
| **Business/Financial success** | Was it financially viable? |
| **Impact on society** | Did it benefit society? |

### Integration Challenges:

| Challenge | Description |
|-----------|-------------|
| High complexity | Many interdependent activities |
| Numerous stakeholders | Conflicting priorities |
| Cross-functional coordination | Multiple departments, contractors |
| Delays in one area affect all | Cascading effects |
| Resource sharing | Limited resources across multiple projects |

### Integration Activities:
- Coordinating sponsors, prime contractors, subcontractors, consultants, suppliers
- Aligning scope, schedule, budget across all parties
- Managing communication and reporting

## Q20. “Success and failure of project depends on project manager” Explain

### Reasons why PM is centerpiece:

| Reason | Explanation |
|--------|-------------|
| **1. Planning and organizing** | PM creates roadmap and structure |
| **2. Leading and motivating teams** | PM inspires team to perform |
| **3. Monitoring progress** | PM tracks against plan and takes corrective action |
| **4. Ensuring delivery** | PM ensures on-time, within-budget completion |
| **5. Communicating with stakeholders** | PM manages expectations |
| **6. Risk management** | PM identifies and mitigates risks |
| **7. Resource allocation** | PM assigns resources optimally |
| **8. Conflict resolution** | PM resolves issues before they escalate |

### Impact on Success:

| PM action | Leads to |
|-----------|----------|
| Clear communication | Alignment, fewer misunderstandings |
| Effective planning | On-time delivery, cost control |
| Strong leadership | High team morale, productivity |
| Proactive risk management | Fewer surprises |
| Stakeholder engagement | Satisfaction, trust |

- Poor coordination → delays, cost overruns
- Lack of leadership → team demotivation, turnover
- Inadequate planning → scope creep, quality issues
- Poor communication → unmet expectations, conflicts

**Conclusion:** Without a skilled PM, even well-designed projects can fail.

## Q21. “Larger the project size, the greater the likelihood that the organization will seek an older, more experienced project manager” Explain

### Reasons:

| Reason | Explanation |
|--------|-------------|
| **1. Real-world problem solving** | Mega projects face unexpected challenges; experienced PMs have proven strategies |
| **2. Leadership under pressure** | Large projects have high stakes, tight deadlines; experience builds resilience |
| **3. Stakeholder management** | Mega projects involve government, clients, vendors, communities; experienced PMs navigate politics |
| **4. Risk and change management** | Big projects prone to scope creep, budget overruns; seasoned PMs anticipate risks |
| **5. Team and resource coordination** | Managing large teams across geographies requires interpersonal skills gained through experience |
| **6. Track record of delivery** | Top management prefers PMs with history of successful completions |

### Why experience > education for mega projects:
- Education provides **theoretical knowledge**, not practical wisdom
- Experience prepares for **real-time decision-making**, human dynamics, political challenges
- Experience comprises **project experience + total experience**

**Note:** Larger project size → older, more experienced project manager sought.

## Q23. “EI-Emotional Intelligence is one of the informal skill/qualification project manager must have as it affects success as well as failure of the project” Explain

### Definition of EI:
> *"The ability to monitor one's own and others' feelings and emotions, to distinguish between them, and to use this information to guide thought and action."* – Salovey & Mayer

### Daniel Goleman's Five Components:

| Component | Description |
|-----------|-------------|
| **Self-awareness** | Recognizing own emotions |
| **Self-regulation** | Managing own emotions |
| **Motivation** | Internal drive |
| **Empathy** | Understanding others' emotions |
| **Social skills** | Managing relationships |

### Why EI is essential for PMs:

| Reason | Explanation |
|--------|-------------|
| **Handling human complexity** | Mega projects involve diverse stakeholders |
| **Leadership beyond formal authority** | EI enables inspiration and motivation |
| **Conflict resolution** | Navigate interpersonal tensions |
| **Decision-making under pressure** | Balance logic with empathy |
| **Team motivation** | Create positive work environment |

### Impact on Project Success/Failure:

| High EI leads to | Low EI leads to |
|-----------------|-----------------|
| Better team collaboration | Poor communication |
| Effective conflict resolution | Unmet expectations |
| Enhanced leadership | Strained relationships |
| Improved stakeholder management | Demotivated team |
| Stress management | Impulsive decisions |

**Conclusion:** While technical skills are essential, EI acts as a key enabler. Low EI can accelerate project failure even when technical planning is strong.

## Q24. Explain use of Tuckman's model of team building in building the project team in project management

### Tuckman's Five Stages:

```
Forming → Storming → Norming → Performing → Adjourning
```

| Stage | Description | Team Behavior | PM's Role |
|-------|-------------|---------------|-----------|
| **1. Forming** | Team meets, introductions, learn project objectives | Polite, anxious, dependent | Directive – provide clear direction, goals |
| **2. Storming** | Conflicts, competition for status, different opinions | Arguments, resistance | Coaching – facilitate, mediate, encourage listening |
| **3. Norming** | Rules accepted, differences respected, trust develops | Cooperation, agreement | Supportive – delegate, build consensus |
| **4. Performing** | High performance, independent decision-making | Synergy, problem-solving | Delegating – monitor, celebrate milestones |
| **5. Adjourning** | Project ends, documentation, celebration | Sadness, recognition | Supportive – acknowledge contributions, capture lessons |

### How Tuckman Model Helps PM:

| Benefit | Explanation |
|---------|-------------|
| **Anticipate team behavior** | Know when to expect conflicts or high performance |
| **Guidance on leadership style** | Each stage demands different leadership (directive → coaching → supportive → delegating) |
| **Supports risk & stakeholder management** | Reduces communication breakdown, role misunderstandings |
| **Enhances team performance** | Accelerates movement toward Performing stage |
| **Improves project success** | Helps build cohesive, effective teams |
